# KL Vistorias - Vistoria Cautelar Premium

![KL Vistorias](https://images.unsplash.com/photo-1487754180451-c456f719a1fc?q=80&w=2070&auto=format&fit=crop)

Este é o repositório oficial da Landing Page da **KL Vistorias**, desenvolvida com tecnologias modernas para garantir alta performance, SEO otimizado e uma experiência de usuário excepcional.

## 🚀 Tecnologias Utilizadas

*   **React + Vite**: Framework ultra-rápido para desenvolvimento frontend.
*   **TypeScript**: Tipagem estática para maior segurança e manutenibilidade.
*   **Tailwind CSS**: Estilização utility-first para um design moderno e responsivo.
*   **Framer Motion**: Animações suaves e interações ricas.
*   **React Helmet Async**: Gerenciamento de SEO (Meta tags, Open Graph, JSON-LD).
*   **Lucide React**: Ícones leves e consistentes.

## ✨ Funcionalidades

*   **Design Responsivo**: Perfeito em celulares, tablets e desktops.
*   **SEO Masterpiece**:
    *   Dados estruturados (Schema.org) para `LocalBusiness` e `FAQPage`.
    *   Open Graph para compartilhamento rico em redes sociais.
    *   Sitemap e Robots.txt configurados.
*   **Performance**:
    *   Lazy loading de imagens.
    *   Preconnect para domínios externos.
    *   PWA (Progressive Web App) instalável.
*   **Conversão**:
    *   Botão flutuante do WhatsApp.
    *   CTAs estratégicos.

## 🛠️ Como Rodar Localmente

1.  Clone o repositório:
    ```bash
    git clone https://github.com/seu-usuario/kl-vistoria.git
    ```
2.  Instale as dependências:
    ```bash
    npm install
    ```
3.  Inicie o servidor de desenvolvimento:
    ```bash
    npm run dev
    ```
4.  Acesse `http://localhost:5173` no seu navegador.

## 📦 Deploy (Vercel)

Este projeto já inclui o arquivo `vercel.json` configurado para SPA.

1.  Crie uma conta na [Vercel](https://vercel.com).
2.  Importe este repositório do GitHub.
3.  O framework "Vite" será detectado automaticamente.
4.  Clique em **Deploy**.

## 📂 Estrutura de Pastas

*   `src/components`: Componentes React modulares.
*   `src/components/ui`: Design System reutilizável.
*   `src/data`: Dados centralizados (informações da empresa).
*   `public`: Arquivos estáticos (favicon, robots.txt, sitemap.xml).
