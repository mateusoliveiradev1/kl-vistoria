# Requirements

## v1 Requirements

### Landing Conversion

- [ ] **LAND-01**: Visitor can understand in the first viewport that KL prevents risky vehicle purchases.
- [ ] **LAND-02**: Visitor can start a WhatsApp scheduling conversation from the hero without scrolling.
- [ ] **LAND-03**: Visitor can quickly see what the inspection checks before opening WhatsApp.
- [ ] **LAND-04**: Visitor can see operational trust signals: mobile service, digital report, local area, CNPJ, and business phone.
- [ ] **LAND-05**: Visitor can reach a second strong CTA after reading proof and objections.

### Visual System

- [ ] **VIS-01**: Page uses a consistent premium visual language across homepage sections.
- [ ] **VIS-02**: Typography hierarchy is readable and not overly decorative on mobile.
- [ ] **VIS-03**: Cards, panels, buttons, and inputs use restrained radii and borders.
- [ ] **VIS-04**: Accent colors are reserved for conversion, trust, and state signals.

### WhatsApp Readiness

- [ ] **BOT-01**: Popup/lead copy asks for the data a future WhatsApp bot needs.
- [ ] **BOT-02**: WhatsApp message text is structured enough for automated parsing later.
- [ ] **BOT-03**: CTAs use direct intent labels instead of vague marketing copy.

## v2 Requirements

- [ ] **BOT-V2-01**: Build the WhatsApp bot flow after the redesigned page is shipped.
- [ ] **CRM-V2-01**: Persist lead qualification into CRM/admin once backend scope is active.

## Out of Scope

- Full WhatsApp automation in Phase 1.
- New backend integration in Phase 1.
- Replacing service-area pages in Phase 1.
